Как открыть сайт без Cursor (чтобы ссылка работала у другого человека)
================================================================

Пока сайт лежит только на вашем компьютере и открывается через Live Server,
другие его не видят. Нужно выложить файлы в интернет (бесплатно).

Самый быстрый способ — Netlify Drop (без программ и без Git)
-------------------------------------------------------------
1. Откройте в браузере: https://app.netlify.com/drop
2. Зайдите в аккаунт Netlify (можно через Google/GitHub) — регистрация бесплатная.
3. Перетащите в окно Netlify ВСЮ папку проекта целиком — в КОРНЕ публикации должны оказаться (как у вас на диске):
   index.html, catalog.html, styles.css, app.js, netlify.toml, favicon.svg, site.webmanifest и каталог images/
   (не создавайте лишний уровень: если архив распаковывается как «папка внутри папки»,
   перетащите внутреннюю папку, где лежит index.html рядом с styles.css).
4. Через ~минуту появится адрес вида: https://случайное-имя.netlify.app
   Эту ссылку можно отправить кому угодно — сайт откроется с любого устройства.

Временный домен «SLT-Узел Тольятти» (канонический URL в коде)
--------------------------------------------------------------
В <head> страниц указан адрес https://slt-uzel-tlt.netlify.app — чтобы ссылки совпадали с реальным
сайтом, в Netlify: Site settings → Domain management → Options → Change site name → введите
slt-uzel-tlt (будет https://slt-uzel-tlt.netlify.app). Если оставите случайное имя — замените
этот URL в index.html и catalog.html (canonical, og:url) на ваш поддомен или будущий домен.
Не забудьте залить favicon.svg в корень публикации вместе с остальными файлами.

Проверка после загрузки (если сайт «без стилей», как простой текст)
-------------------------------------------------------------------
В браузере откройте: https://ваш-поддомен.netlify.app/styles.css
Должен открыться текст таблицы стилей (CSS), а не страница 404.
Если 404 — styles.css не попал в корень сайта; загрузите проект заново, см. п.3 выше.
Так же можно проверить: .../app.js, .../favicon.svg и любую картинку из .../images/home/

Иконка вкладки (favicon): это файл favicon.svg в корне сайта. Страницы уже ссылаются на него в <head>;
отдельно ничего настраивать не нужно — главное, при публикации не выкидывать favicon.svg и site.webmanifest из папки.

В папке уже есть netlify.toml — Netlify сам поймёт, что публиковать.

В настройках сайта на Netlify можно задать красивое имя поддомена или подключить свой домен.

GitHub Pages + Git (обновление одной командой push)
----------------------------------------------------
В проекте настроен workflow: при push в ветку **main** сайт собирается на GitHub Pages.

1. Создайте репозиторий на https://github.com/new (можно без README).
2. В папке проекта на компьютере (один раз):

   git init
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
   git add -A
   git commit -m "Первый коммит сайта"
   git push -u origin main

3. На GitHub: Settings → **Pages** → **Build and deployment** → Source: **GitHub Actions**
   (не «Deploy from a branch»). Сохраните.
4. Settings → **Actions** → **General** → раздел *Workflow permissions*:
   поставьте **Read and write permissions** (нужно, чтобы GITHUB_TOKEN мог публиковать Pages).
5. Сайт будет по адресу: **https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/**

Если в разделе Environments (Deployments) environment **github-pages** с красным крестиком:
   • Откройте **Actions** → последний **failed** run → внизу прочитайте текст ошибки.
   • Settings → **Environments** → **github-pages** — отключите *Required reviewers*, если
     деплой «висит» в ожидании одобрения.
   • Убедитесь, что в п.3 выбран именно **GitHub Actions**, затем: Actions →
     **Re-run all jobs** для последнего workflow.

Дальше при любых правках:

   git add -A
   git commit -m "Кратко, что изменили"
   git push

Через 1–2 минуты обновится опубликованная версия.

В index.html и catalog.html в canonical / og:url сейчас указан Netlify — при желании замените на
URL GitHub Pages или на свой домен (и подтвердите сайт в FormSubmit, если меняется адрес формы).

Альтернативы
-----------
• Cloudflare Pages: https://pages.cloudflare.com/ — загрузка или Git.
• Любой хостинг с «статикой» / FTP: загрузить все файлы из папки, сохраняя структуру
  (в т.ч. favicon.svg, site.webmanifest, images/ и т.д.).

Важно
-----
• Сайт не тянет шрифты с Google — страницы открываются и на мобильном интернете, где fonts.googleapis.com
  часто тормозит или недоступен.
• После смены домена в FormSubmit зайдите в письмо от сервиса и при необходимости
  подтвердите новый адрес сайта, чтобы форма заявок продолжила работать.
• Не выкладывайте в публичный репозиторий секретные ключи (у вас в форме — id FormSubmit).
